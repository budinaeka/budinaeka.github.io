## 🎯 Decision Flowchart

```
START: What is your project?
│
├─ Is it published in a journal?
│  └─ YES → 🟢 research
│
├─ Is it a map or GIS?
│  ├─ Interactive web map? → 🔷 webgis
│  ├─ Risk/disease map? → 🔵 mapping
│  └─ Disaster related? → 🟠 disaster
│
├─ Does it use AI/ML?
│  ├─ Uses LLM (GPT, Claude)? → 🟪 genai
│  ├─ Autonomous bot/agent? → 🔮 aiagents
│  ├─ Traditional ML model? → 🟣 ml
│  └─ Creative/experimental? → 🎀 vibecoding
│
└─ Something else? → ⚪ other
```

## 🎯 Common Scenarios

### Scenario 1: "I built a chatbot using GPT-4"
```
Question: Is it autonomous or just responds?
├─ Just responds → 🟪 genai
└─ Autonomous actions → 🔮 aiagents
```

### Scenario 2: "I made an interactive disease map"
```
Question: What's the main feature?
├─ Risk analysis → 🔵 mapping
├─ Web interaction → 🔷 webgis
└─ Creative viz → 🎀 vibecoding
```

### Scenario 3: "I trained a prediction model"
```
Question: What type of model?
├─ Traditional ML → 🟣 ml
├─ LLM-based → 🟪 genai
└─ Agent-based → 🔮 aiagents
```

### Scenario 4: "I created an animated dashboard"
```
Question: What's the focus?
├─ Data accuracy → 🔷 webgis
├─ Aesthetics → 🎀 vibecoding
└─ AI features → 🟪 genai
```

## 💡 Pro Tips

### ✅ DO:
- Choose based on PRIMARY innovation
- Consider your target audience
- Think about what you want to showcase
- Match category to project strength

### ❌ DON'T:
- Mix multiple categories (pick one!)
- Choose based on tech stack alone
- Ignore the visual/aesthetic aspect
- Overthink it (you can always change)
